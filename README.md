# ระบบติดตามแผนงานโครงการ โรงเรียนบ้านวังกระดี่ทอง (Executive Dashboard)

เว็บไซ์ Dashboard ระบบรายงานและติดตามแผนงานโครงการสำหรับผู้บริหาร โรงเรียนบ้านวังกระดี่ทอง ออกแบบและพัฒนาด้วยดีไซน์ที่หรูหรา (Premium Dark/Light Glassmorphism Theme) เพื่อแสดงผลข้อมูลแบบเรียลไทม์ผ่านการเชื่อมต่อข้อมูลกับ Google Sheets โดยตรง

## 🌟 คุณสมบัติเด่น (Features)
1. **การโหลดข้อมูลแบบ Live Sync**: ดึงข้อมูลและคำนวณใหม่ทุกครั้งที่เข้าหน้าเว็บ หรือกดปุ่ม "รีเฟรชข้อมูล" โดยไม่มีการทำแคชข้อมูลเก่าไว้ (Cache-Busting)
2. **ระบบวิเคราะห์ข้อมูล (Analytical Charts)**:
   - แผนภูมิโดนัทวิเคราะห์สัดส่วนสถานะการดำเนินงาน
   - แผนภูมิแท่งเปรียบเทียบงบประมาณแบ่งตามกลุ่มงาน (งบจัดสรร vs. งบเบิกจ่าย vs. งบคงเหลือ)
3. **ระบบการกรองข้อมูลอัจฉริยะ (Smart Filters)**: ค้นหาคำสำคัญในโครงการหรือผู้รับผิดชอบ, กรองแยกกลุ่มงาน, หรือกรองตามสถานะความคืบหน้าได้ทันที
4. **สรุปสถิติวัดผลหลัก (KPI Overview)**:
   - จำนวนโครงการทั้งหมด
   - งบประมาณรวมที่ได้รับการจัดสรร
   - งบประมาณใช้ไปแล้ว (เบิกจ่าย) และเปอร์เซ็นต์สะสม
   - งบประมาณคงเหลือพร้อมใช้ และเปอร์เซ็นต์
   - ความคืบหน้าภาพรวมของโรงเรียน (คำนวณเฉลี่ยความคืบหน้าของโครงการ)
5. **รายละเอียดรายโครงการ (Detailed Project Modal)**: ดับเบิลคลิกหรือกดที่แถวของโครงการเพื่อดูสรุปสัดส่วนงบประมาณเป็นกราฟจำลองสวยงามพร้อมข้อมูลผู้รับผิดชอบ
6. **การสลับสว่าง-มืด (Dual Theme)**: รองรับทั้ง Dark Mode และ Light Mode เพื่อความสบายตา

## 📊 แหล่งข้อมูล (Data Source)
ระบบนี้ดึงข้อมูลจาก Google Sheets สาธารณะโดยตรงที่ลิงก์:
`https://docs.google.com/spreadsheets/d/1zr741Uz6KacAdAySOfy5Wqymm1J7vaeXG3dy7gV0xUY/edit`

โครงสร้างคอลัมน์ของตารางข้อมูลที่ดึงมาคำนวณ:
1. **รหัสโครงการ**: รหัสประจำตัวโครงการ
2. **ชื่อโครงการ**: ชื่อและหัวข้อโครงการ
3. **ผู้รับผิดชอบ**: ครูผู้ดำเนินการโครงการ
4. **กลุ่มงาน**: ฝ่าย/กลุ่มงาน เช่น วิชาการ, วิทยาศาสตร์และเทคโนโลยี, สังคมศึกษา, งบประมาณและสินทรัพย์
5. **งบประมาณ**: จำนวนเงินงบประมาณเต็มโครงการ
6. **ใช้ไปแล้ว**: จำนวนงบประมาณที่ได้เบิกจ่ายดำเนินงานแล้ว
7. **คงเหลือ**: จำนวนเงินเหลือพร้อมใช้
8. **ความคืบหน้า**: เปอร์เซ็นต์ความคืบหน้า (0-100)
9. **สถานะ**: ระบุ "ยังไม่ดำเนินการ", "อยู่ระหว่างดำเนินการ", หรือ "ดำเนินการแล้ว"

## 🚀 การอัปโหลดหน้าเว็บขึ้นเผยแพร่บน GitHub (Deployment Guide)

เนื่องจากเว็บนี้พัฒนาด้วยภาษา HTML, CSS และ JavaScript แท้ (Vanilla JS) โดยดึงข้อมูลแบบไดนามิกจากทางผู้ใช้โดยตรง จึงสามารถทำงานและโฮสต์บน **GitHub Pages** ได้ฟรี 100% โดยไม่ต้องใช้เครื่องคอมพิวเตอร์เซิร์ฟเวอร์ใดๆ

ทำตามขั้นตอนด้านล่างเพื่อทำการเผยแพร่เว็บไซต์:

### ขั้นตอนที่ 1: เตรียมโค้ดของคุณ
ตรวจสอบให้แน่ใจว่าในโฟลเดอร์โครงการของคุณมีไฟล์ดังต่อไปนี้อยู่ครบถ้วน:
- `index.html`
- `style.css`
- `app.js`
- `README.md`

### ขั้นตอนที่ 2: สร้าง Repository บน GitHub
1. เข้าไปที่ [GitHub](https://github.com/) และเข้าสู่ระบบบัญชีของคุณ
2. กดปุ่ม **New** หรือเครื่องหมาย **+** ที่มุมขวาบนเพื่อสร้างคลังเก็บโค้ด (Repository) ใหม่
3. ตั้งชื่อ Repository (เช่น `school-project-dashboard`)
4. ตรวจสอบให้มั่นใจว่าตั้งค่าเป็น **Public** (สาธารณะ) เพื่อให้ GitHub Pages ทำงานได้
5. **ห้าม** ติ๊กเลือก Add a README file, Add .gitignore หรือ Choose a license (ปล่อยว่างไว้ทั้งหมด)
6. กดปุ่ม **Create repository**

### ขั้นตอนที่ 3: อัปโหลดโค้ดด้วย Git (หรือผ่านเว็บบราวเซอร์)

**วิธีอัปโหลดผ่าน Git Command Line:**
เปิดโปรแกรม Terminal หรือ Git Bash ในเครื่องคอมพิวเตอร์ของคุณที่โฟลเดอร์นี้ แล้วพิมพ์คำสั่งดังนี้ทีละบรรทัด:

```bash
# 1. เริ่มต้นระบบควบคุมเวอร์ชัน
git init

# 2. เพิ่มไฟล์ทั้งหมดเข้าสู่คลังจัดเตรียม
git add .

# 3. บันทึกข้อมูลเวอร์ชันแรก
git commit -m "Initial executive dashboard deployment"

# 4. เปลี่ยนชื่อสาขาหลักเป็น main
git branch -M main

# 5. เชื่อมโยงกับคลังข้อมูลบน GitHub (แก้ไข URL เป็นลิงก์โปรเจกต์ของคุณ)
git remote add origin https://github.com/<บัญชีผู้ใช้ของคุณ>/school-project-dashboard.git

# 6. อัปโหลดไฟล์ขึ้น GitHub
git push -u origin main
```

**หรือวิธีอัปโหลดทางเว็บบราวเซอร์โดยตรง (ไม่ต้องใช้โปรแกรม):**
1. ในหน้า Repository ที่สร้างเสร็จบนเว็บ GitHub ให้กดที่ข้อความลิงก์ **"uploading an existing file"**
2. ลากไฟล์ `index.html`, `style.css`, `app.js`, `README.md` ทั้ง 4 ไฟล์มาใส่ในช่องอัปโหลดหน้าเว็บ
3. รอจนอัปโหลดไฟล์เสร็จสิ้น เลื่อนลงมาด้านล่างแล้วกดปุ่ม **Commit changes**

### ขั้นตอนที่ 4: เปิดใช้งานระบบ GitHub Pages
1. ในหน้าโปรเจกต์บน GitHub ให้คลิกที่แท็บเมนู ⚙️ **Settings** (การตั้งค่า)
2. เมนูด้านซ้ายมองหาหัวข้อ **Code and automation** แล้วคลิกเลือก 📑 **Pages**
3. ภายใต้หัวข้อ **Build and deployment** ให้เลือกตั้งค่าดังนี้:
   - **Source**: `Deploy from a branch`
   - **Branch**: เลือก `main` และโฟลเดอร์ `/ (root)`
4. กดปุ่ม 💾 **Save**
5. รอระบบของ GitHub ประมวลผลประมาณ 1-2 นาที จากนั้นให้ลองรีเฟรชหน้าเว็บนั้น
6. จะมีกรอบแสดงลิงก์เว็บไซต์สีเขียวแสดงขึ้นมาที่ด้านบนสุด เช่น:
   👉 `https://<ชื่อผู้ใช้ของคุณ>.github.io/school-project-dashboard/`

เมื่อผู้บริหารเข้าใช้งานผ่านลิงก์นี้ ตัวบราวเซอร์จะดึงข้อมูลอัปเดตโครงการล่าสุดจาก Google Sheet มาสรุปผลแบบเรียลไทม์ทันทีโดยไม่ต้องอัปโหลดไฟล์หน้าเว็บขึ้นไปใหม่!
